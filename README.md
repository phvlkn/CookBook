# CookBook Project

## 1. Описание

CookBook — учебный проект, разделённый на несколько веток для совместной работы над фронтендом, бэкендом и документацией.  
Каждый участник работает в своей ветке, не внося изменения напрямую в `main`.

## 2. Клонирование репозитория

Чтобы скопировать репозиторий локально:

```bash
git clone https://github.com/phvlkn/CookBook.git
cd CookBook
```

Если папка уже существует, можно инициализировать репозиторий внутри неё:

```bash
git init
git remote add origin https://github.com/phvlkn/CookBook.git
git pull origin main
```

## 3. Работа с ветками

Каждый участник работает в своей ветке.
Пример структуры веток:

| Ветка    | Назначение                        |
|----------|-----------------------------------|
| main     | стабильная версия проекта         |
| front    | разработка фронтенда              |
| back     | разработка бэкенда                |
| docs     | документация проекта              |
| server   | инфраструктура, Docker и окружение |

Проверка текущей ветки:

```bash
git branch
```

Переход в свою ветку:

```bash
git checkout <название_ветки>
```

Пример:

```bash
git checkout front
```

Если ветка ещё не создана:

```bash
git checkout -b <название_ветки>
```

## 4. Внесение изменений

После внесения правок:

```bash
git add .
git commit -m "описание изменений"
git push origin <твоя_ветка>
```

Пример:

```bash
git add .
git commit -m "добавлен компонент Header"
git push origin front
```

## 5. Получение обновлений

Перед началом работы рекомендуется обновить свою ветку:

```bash
git pull origin <твоя_ветка>
```

Если нужно подтянуть изменения из main:

```bash
git fetch origin
git merge origin/main
```

## 6. Правила работы с main

* В main не пушить напрямую
* Все изменения добавляются через Pull Request (PR)
* Перед созданием PR убедитесь, что ветка обновлена относительно main

Пример обновления ветки:

```bash
git pull origin main
git merge main
```

## 7. Проверка состояния и логов

```bash
git status          # показывает изменения
git log --oneline   # краткий журнал коммитов
git diff            # сравнить изменения
git branch -a       # список локальных и удалённых веток
```

## 8. Общие рекомендации

* Всегда выполняйте git pull перед началом работы
* Делайте небольшие, логичные коммиты с понятными сообщениями
* Проверяйте, что проект собирается локально перед пушем
* В main добавляются только проверенные и стабильные версии
